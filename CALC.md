# Анализ функций чтения из карт в calc.cpp

В файле есть несколько функций для интерполяции значений из 2D-таблиц (карт):

getValFromMap — базовая функция интерполяции
getIgnAdv — угол опережения зажигания
getInjPw — длительность импульса форсунки
И другие, использующие те же карты

Основной алгоритм — **билинейная интерполяция**

Структура карты

Карта представляет собой 2D-таблицу с осями:
X — обычно RPM (обороты двигателя)
Y — нагрузка (MAP, TPS и т.п.)
        RPM:  800  1200  1600  2000 ...
Load:
  20%  [ v00   v10   v20   v30 ]
  40%  [ v01   v11   v21   v31 ]
  60%  [ v02   v12   v22   v32 ]

Пошаговый алгоритм getValFromMap

Шаг 1 — Найти индексы по оси X

```cpp
// Ищем между какими узлами сетки находится текущее значение X
while (x > xAxis[xIdx + 1] && xIdx < xSize - 2)
    xIdx++;
```

Находим xIdx такой, что:

xAxis[xIdx] <= x <= xAxis[xIdx + 1]

Шаг 2 — Найти индексы по оси Y

```cpp
while (y > yAxis[yIdx + 1] && yIdx < ySize - 2)
    yIdx++;
```

Аналогично для Y.

Шаг 3 — Вычислить коэффициенты интерполяции

```cpp
// Насколько X "продвинулся" между узлами (0.0 ... 1.0)
float xFrac = (x - xAxis[xIdx]) / (xAxis[xIdx+1] - xAxis[xIdx]);

// Аналогично для Y
float yFrac = (y - yAxis[yIdx]) / (yAxis[yIdx+1] - yAxis[yIdx]);
```

Шаг 4 — Билинейная интерполяция

```text
    xIdx    xIdx+1
yIdx    v00 ---- v10
         |       |
         |   *   |      * = искомая точка
         |       |
yIdx+1  v01 ---- v11
```

```cpp
// Интерполяция по X для верхней строки
float top = v00 + xFrac * (v10 - v00);

// Интерполяция по X для нижней строки  
float bot = v01 + xFrac * (v11 - v01);

// Интерполяция по Y между результатами
float result = top + yFrac * (bot - top);
```

Визуально — что происходит

Если RPM = 1400, Load = 30%:

        1200    1600
  20%  [ 10  |  14 ]   → top = 10 + 0.5*(14-10) = 12
        -----+-----
  40%  [ 18  |  22 ]   → bot = 18 + 0.5*(22-18) = 20
  
xFrac = (1400-1200)/(1600-1200) = 0.5
yFrac = (30-20)/(40-20) = 0.5

result = 12 + 0.5*(20-12) = 16

Граничные случаи

| Ситуация | Поведение |
|----------|-----------| 
|X < min(xAxis) | Используется первый столбец |
|X > max(xAxis) | Используется последний столбец |
|Y < min(yAxis) | Используется первая строка |
Y > max(yAxis) | Используется последняя строка |

Значения не экстраполируются, а зажимаются на границах таблицы.

Итог

Алгоритм — классическая билинейная интерполяция по прямоугольной сетке:
- Найти 4 ближайших узла вокруг точки
- Интерполировать по X дважды (верх и низ)
- Интерполировать по Y между результатами

Это стандартный подход в ECU (блоках управления двигателем) — позволяет хранить компактную таблицу и получать плавные значения между узлами.
